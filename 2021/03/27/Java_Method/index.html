一、什么是Method

从外面看，方法可以理解成一个小黑盒（一个System）；

“什么是System呢？好奇的同学可以去学习一下神奇的《信号和系统》或者简单理解就是一个小黑盒就可以啦！”

我们从外部给了它一些适当的参数，它处理完之后返回给我们一个或者多个正确的值

就像下面这张图：


二、为什么要用它

  1. 直观、简化：将原来一大串的代码分散，条理清晰；

    “大道至简，分而治之”

  2. 便于维护：“不会有人debug的时候想从10000+行的代码里找到有问题的变量吧，不会吧，不会吧。”

  3. 减少冗余代码：”但凡每个月都是28天，我要写这80行代码？——某位成功判断闰年的同学“



三、Method的使用

（一）声明

声明方法时需要注意的几点

  1. 方法名
  2. 传入参数
  3. 返回参数
  4. 是否静态
  5. 可被使用的范围
  6. Your coding

（二）传入参数

注意辨别形式参数和实例参数

形参： 声明方法时用定义的一个变量的名字。

“因为你写方法的时候不知道会有什么数值进来，所以需要一个代号代表他们——未知数 X”


实参： 外部调用该方法时，给形参一个确定的值。

“现在,你知道这个变量是啥了”


（三）返回值

返回值和return是好兄弟，什么事都要一起做；

——返回值定义为int的时候 return value（int类型的数）；

——返回值定义为int[]**的话 **return value （int[] 类型的数组）；

当返回值溜走玩的时候（为void），return 也可以省略；

“执行return后方法就结束了！”
“执行return后方法就结束了！”
“执行return后方法就结束了！”

（四）方法重载（Overload）

一个方法的名字并不能唯一确定一个方法，就像你和与你异父异母却同名同姓的亲兄弟一样；

但是身份证号可以，它是唯一的，不会重复的；

那么在Java里， 一个方法的身份证号就是它的方法名字 + 传入的参数，这是唯一的。

需要注意的一点是，传入的参数部分，不同数量、不同类型的传入参数的组合，再加上方法名，都可以成为一个唯一的身份证号。


四、方法调用栈

（一）方法的调用/返回顺序

“哆啦A梦想偷懒，所以喊了B梦…….”

A调用B，B调用C，C调用D，D解决问题给C，C给B，B给A；

就像串糖葫芦和吃糖葫芦的顺序，后串上的先被吃；
